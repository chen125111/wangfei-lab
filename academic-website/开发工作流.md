## 正确的开发流程

### 1. 开发阶段（在 GitHub 上进行）
```bash
# 克隆仓库到本地开发环境
git clone https://github.com/chen125111/wangfei-lab.git
cd wangfei-lab

# 创建功能分支
git checkout -b feature/新功能名称

# 进行修改
# ... 编辑文件 ...

# 提交更改
git add .
git commit -m "添加新功能"

# 推送到 GitHub
git push origin feature/新功能名称

# 在 GitHub 上创建 Pull Request
```

### 2. 测试阶段
- 在 GitHub 上 review Pull Request
- 确保代码质量
- 测试功能是否正常

### 3. 部署阶段（确认合适后）
```bash
# 在服务器上拉取最新代码
cd /var/www/wangfei
git pull origin main

# 重启服务（如果需要）
systemctl restart nginx
```

## 重要规则
1. **不要直接修改服务器上的文件** - 这样改动会立即暴露给用户
2. **所有开发在 GitHub 分支上进行** - 创建分支 → 修改 → PR → 合并
3. **确认合适后再部署** - 只有经过测试的代码才部署到服务器
4. **保持版本控制** - 每次部署都有明确的提交记录
